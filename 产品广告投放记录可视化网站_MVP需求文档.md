# 产品广告投放记录可视化网站  
## MVP 需求文档

**版本：** MVP V1.0  
**文档日期：** 2026-07-16  
**目标用户：** 广告投放团队  
**访问方式：** 固定网址公开访问，无登录、无访问密码  

---

## 1. 项目背景

团队每天会从广告平台导出某个产品款式对应的广告素材数据，并以 Excel 或 CSV 文件形式保存。

当前需要建立一个简单的广告投放数据管理与可视化网站，实现：

1. 按产品款式管理广告日报。
2. 每日上传单日广告数据。
3. 长期保存每日明细及原始文件。
4. 自动按作品 ID 汇总素材的整体表现。
5. 查看单条素材在投放期间的每日趋势。
6. 支持多人通过同一个固定网址查看和管理相同数据。

---

## 2. MVP 核心目标

MVP 首版重点解决以下问题：

- 不再手工复制、粘贴和汇总每日广告数据。
- 自动识别不同日期中重复出现的同一作品 ID。
- 自动计算每条素材的总成本、总收入、ROI、CPM 等指标。
- 快速筛选和排序低效或高消耗素材。
- 点击单条素材即可查看投放期间的趋势图和每日明细。
- 保留历史日报和原始文件，便于后续核查。

## 4. 核心业务概念

### 4.1 产品

一个产品代表一个广告投放款式。

MVP 中产品仅保存：

- 产品名称
- 创建时间
- 产品状态
- 归档时间

产品状态：

- 正常
- 已归档

产品归档后：

- 不在默认产品首页展示。
- 历史日报和广告数据继续保留。
- 支持恢复归档产品。
- 修改产品名称不影响历史广告数据。

---

### 4.2 日报

一份日报文件只属于一个产品，并代表一个日期的单日广告数据。

上传流程：

1. 用户先选择产品。
2. 用户上传 Excel、XLS 或 CSV 文件。
3. 系统读取文件名中的日期。
4. 系统自动寻找包含“作品 ID”字段的工作表。
5. 系统校验必需字段。
6. 校验成功后保存数据及原始文件。
7. 系统重新计算产品和素材汇总数据。
8. 页面显示“上传成功”。

上传失败时，需要显示具体失败原因。

---

### 4.3 素材

素材以“作品 ID”作为主要识别字段。

规则：

- 同一产品下，相同作品 ID 在不同日期出现时，视为同一条素材。
- 每一份单日日报中，同一作品 ID 只出现一行。
- 同一作品 ID 可在多天日报中重复出现。
- 作品 ID 为 `N/A` 时，统一汇总为该产品下的一条“商品卡片”素材。
- 不同产品下的 `N/A` 分别统计，不跨产品合并。

## 5. 数据文件要求

### 5.1 支持格式

MVP 支持：

- `.xlsx`
- `.xls`
- `.csv`

三种格式使用相同的字段校验规则。

---

### 5.2 文件日期规则

建议文件名采用：

```text
YYYY-MM-DD.xlsx
YYYY-MM-DD.xls
YYYY-MM-DD.csv
```

示例：

```text
2026-07-16.xlsx
```

规则：

- 系统优先从文件名读取日报日期。
- 文件日期允许晚于当前日期。
- 文件名格式不正确时，需要给出提醒。
- 为避免无法保存日报，文件名无法解析日期时，应允许用户手动输入或确认日报日期后继续上传。

---

### 5.3 工作表识别

当 Excel 文件包含多个工作表时：

- 系统自动寻找包含“作品 ID”表头的工作表。
- 找到一个符合条件的工作表后读取该表。
- 未找到符合条件的工作表时，上传失败。
- 如果找到多个符合条件的工作表，MVP 可默认读取第一个，并给出提示。

CSV 文件直接读取其唯一数据表。

---

### 5.4 必需字段

上传文件必须包含当前广告报表中的以下全部 23 个字段：

1. 作品 ID
2. 创意素材
3. TikTok 账号
4. 创意作品类型
5. 视频来源
6. 状态
7. 发布时间
8. 成本
9. SKU 订单数
10. 平均下单成本
11. 总收入
12. ROI
13. 商品广告曝光数
14. 商品广告点击数
15. 商品广告点击率
16. 广告转化率
17. 2秒播放率
18. 6秒播放率
19. 25%播放率
20. 50%播放率
21. 75%播放率
22. 完播率
23. 货币

字段校验规则：

- 根据表头名称识别字段，不要求列顺序固定。
- 缺少任意必需字段时，整份文件上传失败。
- 文件中出现额外字段时自动忽略。
- 不因列顺序变化而上传失败。
- 数值字段为空时按 `0` 保存。
- 货币固定为 USD。
- 当前不进行汇率换算。

---

## 6. 同日重复上传规则

同一个产品、同一个日报日期允许重复上传。

处理方式：

1. 每次上传都保存为一个独立版本。
2. 原始文件和解析后的数据均保留。
3. 同一产品、同一日期只使用最新上传版本参与统计。
4. 旧版本保留，但 MVP 前台不提供历史版本查看和恢复入口。
5. 日报记录页显示当前生效版本。
6. 删除最新生效版本后，系统可将该日期剩余版本中最新的一份自动设为生效版本。
7. 如果该日期不存在其他版本，则该日期从统计中移除。

---

## 7. 数据保存原则

每条日报明细需要保存：

- 所属产品
- 日报日期
- 上传批次
- 是否为当前生效版本
- 原始文件名
- 原始文件地址
- 上传时间
- 23 个原始广告字段
- 系统计算的 CPM
- 数据创建时间

原始 Excel、XLS 或 CSV 文件需要长期保留，并可在日报记录页下载。

上传后的广告明细不允许手动修改。

发现数据错误时，用户需要重新上传同一产品、同一日期的正确文件，由最新版本覆盖统计结果。

## 8. 指标计算规则

### 8.1 每日明细指标

每日明细中的以下字段直接展示文件原始值：

- 成本
- SKU 订单数
- 总收入
- ROI
- 商品广告曝光数
- 商品广告点击数
- 商品广告点击率
- 广告转化率
- 各播放率指标

每日 CPM 由网站计算：

```text
每日 CPM = 每日成本 ÷ 每日商品广告曝光数 × 1000
```

异常处理：

- 曝光数为 0 时，CPM 显示为 `—` 或空值，不能报错。
- 金额统一显示美元符号 `$`。
- ROI 按倍数展示，不显示百分号。
- 点击率、转化率和播放率按百分比展示。

---

### 8.2 素材汇总指标

同一产品下，根据作品 ID 汇总所有生效日报数据。

#### 总成本

```text
总成本 = 各日报成本之和
```

#### 总收入

```text
总收入 = 各日报总收入之和
```

#### 总订单数

```text
总订单数 = 各日报 SKU 订单数之和
```

#### 总曝光数

```text
总曝光数 = 各日报商品广告曝光数之和
```

#### 总点击数

```text
总点击数 = 各日报商品广告点击数之和
```

#### 总 ROI

```text
总 ROI = 总收入 ÷ 总成本
```

总成本为 0 时，显示 `—`。

#### 总点击率

```text
总点击率 = 总点击数 ÷ 总曝光数
```

总曝光数为 0 时，显示 `—`。

#### 总转化率

```text
总转化率 = 总订单数 ÷ 总点击数
```

总点击数为 0 时，显示 `—`。

#### 总 CPM

```text
总 CPM = 总成本 ÷ 总曝光数 × 1000
```

总曝光数为 0 时，显示 `—`。

注意：

- 汇总 ROI 不能使用每日 ROI 的平均值。
- 汇总点击率不能使用每日点击率的平均值。
- 汇总转化率不能使用每日转化率的平均值。
- 汇总 CPM 不能使用每日 CPM 的平均值。

## 9. 素材时间与状态规则

### 9.1 发布时间

- 直接保存并展示 Excel 中的原始发布时间。
- 不进行时区转换。

### 9.2 投放起始时间

```text
投放起始时间 = 作品 ID 第一次出现在生效日报中的日期
```

视频或素材只要出现在日报中，就代表已经开始投放，不要求成本大于 0。

### 9.3 投放最新更新时间

```text
投放最新更新时间 = 作品 ID 最近一次出现在生效日报中的日期
```

### 9.4 投放天数

```text
投放天数 = 投放最新更新时间 - 投放起始时间 + 1
```

按自然天数计算。

示例：

- 7 月 1 日首次出现
- 7 月 2 日未出现
- 7 月 3 日再次出现

投放天数为 3 天。

### 9.5 当前状态

状态处理规则：

1. 如果素材出现在产品最新一份生效日报中，则使用该日报中的原始状态。
2. 如果素材未出现在产品最新一份生效日报中，则当前状态自动标记为“已排除”。
3. 如果已排除素材后续重新出现在新的日报中，则自动恢复为新日报中的原始状态。
4. Excel 中出现新的状态值时，原样保存，不做固定枚举转换。

### 9.6 某天未出现

当某素材在某天的已上传完整日报中未出现：

- 该素材当天各项广告数据按 0 处理。
- 当天状态视为“已排除”。
- 该日期仍计入首次出现日至最新出现日之间的投放自然天数。

当某个日期根本没有上传日报时：

- 该日期视为缺失数据。
- 不自动补 0。
- 趋势图保留日期位置，但折线断开。
- 不将“未上传日报”误判为“素材当天没有投放”。

### 删除日报

删除后：

1. 删除该上传版本的数据。
2. 如果同产品、同日期存在其他版本，则自动选择剩余版本中最新的一份作为生效版本。
3. 如果不存在其他版本，则移除该日期数据。
4. 自动重新计算产品汇总。
5. 自动重新计算素材汇总。
6. 自动重新计算状态、投放时间和投放天数。
7. 自动刷新趋势图和每日明细。

MVP 不提供同日历史版本查看或恢复功能。

## 12. 删除与操作记录

产品归档、产品恢复和日报删除需要：

- 二次确认

- 保存操作对象
- 保存操作前后的关键状态

由于网站无登录系统，MVP 无法准确识别具体操作人，可统一记录为：

```text
public-user
```

操作记录至少包括：

- 新增产品
- 修改产品名称
- 归档产品
- 恢复产品
- 上传日报
- 删除日报

## 13. 数据展示格式

### 13.1 金额

统一使用美元：

```text
$1,234.56
```

适用字段：

- 成本
- 总收入
- 平均下单成本
- CPM

### 13.2 数量

整数千分位：

```text
12,345
```

适用字段：

- 订单数
- 曝光数
- 点击数
- 素材数

### 13.3 ROI

建议保留两位小数：

```text
2.58
```

```text

2026-07-16
```

发布时间：

```text
2026-07-16 10:30:25
```

## 14. 空值与异常值处理

- 数值字段为空：按 0 保存。
- 文本字段为空：保存为空字符串或 `—`。
- ROI 分母为 0：显示 `—`。
- 点击率分母为 0：显示 `—`。
- 转化率分母为 0：显示 `—`。
- CPM 分母为 0：显示 `—`。
- 作品 ID 为空但不等于合法的 `N/A`：上传失败或标记为异常。
- 完全重复的数据行：由于同一日报内作品 ID 唯一，检测到重复作品 ID 时整份文件上传失败。
- 负数金额或数量：原样保存，但建议记录异常日志，不自动修改。

---

## 15. 数据一致性要求

1. 每个产品、日报日期只能有一个生效版本。
2. 汇总只读取生效版本。
3. 修改产品名称不改变历史数据归属。
4. 删除日报后必须重新计算，不能只删除文件。
5. 产品归档不删除日报和素材。
6. 缺少整日文件时不能补 0。
7. 素材仅在已上传完整日报中未出现时才按 0 和“已排除”处理。
8. 所有汇总比率必须通过汇总后的分子、分母重新计算。
9. 每日比率展示原文件值。
10. 原始文件与解析数据需要可关联追溯。

## 16. 非功能需求

### 16.1 使用方式

- 固定网址访问。
- 无登录。
- 无访问密码。
- 所有人均可查看、上传、删除和归档。
- 所有访问者看到同一份云端数据。

### 16.2 性能

MVP 建议目标：

- 产品首页加载时间不超过 3 秒。
- 素材汇总表翻页响应不超过 2 秒。
- 单条素材趋势弹窗打开不超过 2 秒。
- 单个文件建议支持至少 10,000 行。
- 上传时提供加载状态，避免重复提交。

### 16.3 浏览器

优先支持：

- Chrome 桌面版
- Edge 桌面版

移动端仅保证基本查看，不要求首版完成复杂表格操作适配。

### 16.4 数据安全说明

当前需求明确选择：

- 网站完全公开。
- 无登录和访问密码。
- 所有人均可上传、删除和归档。

因此存在以下已知风险：

- 任何获得网址的人都能查看广告数据。
- 任何获得网址的人都能上传错误文件。
- 任何获得网址的人都能删除日报或归档产品。
- 操作记录无法确认真实操作者身份。

MVP 可按此要求实现，但建议后续版本至少增加团队共用管理密码。

## 17. 推荐 MVP 技术方案

为实现多人共享、云端长期保存和原始文件下载，无法只使用浏览器本地存储。

推荐采用“前端项目 + 托管云服务”，不开发独立后端服务器：

### 前端

- React + Vite
- Tailwind CSS 或简洁 CSS
- Recharts
- SheetJS

### 数据及文件

- Supabase PostgreSQL
- Supabase Storage

### 部署

- Vercel 或 Netlify

### 说明

该方案仍然保持开发简单：

- 只有一个前端代码项目。
- 无需维护传统后端服务器。
- 数据库和文件存储由 Supabase 托管。
- 所有用户通过同一个网址访问同一数据。

由于网站无登录，Supabase 数据库和文件操作需要通过受控服务接口或 Edge Function 处理，避免直接暴露高权限密钥。

## 19. MVP 验收标准

### 产品管理

- 可以新增产品。
- 可以修改产品名称。
- 可以归档产品。
- 可以恢复产品。
- 归档不会删除历史数据。

### 文件上传

- 可以选择产品后上传 XLSX、XLS、CSV。
- 可以自动读取包含“作品 ID”的工作表。
- 可以按表头名称识别 23 个字段。
- 列顺序变化不影响上传。
- 缺字段时上传失败并显示原因。
- 额外字段被忽略。
- 空数值按 0 保存。
- 原始文件可保存和下载。
- 同日重复上传后最新版本生效。

### 汇总

- 相同作品 ID 可以跨日报正确汇总。
- `N/A` 可以汇总为“商品卡片”。
- 总 ROI、点击率、转化率、CPM 计算正确。
- 当前状态规则正确。
- 投放时间和投放天数计算正确。
- 缺少整日文件时不补 0。
- 已上传日报中未出现的素材按 0 和“已排除”处理。

### 页面

- 产品首页可以查看产品卡片。
- 产品详情页显示整体汇总。
- 素材表每页 20 条并可翻页。
- 表头和作品 ID 列可固定。
- 可以按状态筛选。
- 可以搜索作品 ID。
- 可以按成本、收入、ROI、CPM 排序。
- 点击素材可以打开趋势弹窗。
- 趋势图可以切换四个指标。
- 弹窗中可以查看每日明细。
- 日报记录页可以下载和删除日报。

### 删除与重算

- 删除日报需要二次确认。
- 删除后所有相关汇总自动更新。
- 删除后素材状态和生命周期自动更新。
- 同日存在旧版本时可自动切换生效版本。

## 22. 最终确认结论

本 MVP 采用以下核心原则：

- 一个文件对应一个产品和一个日报日期。
- 上传前手动选择产品。
- 每日上传单日数据，不上传累计数据。
- 所有日报长期保留。
- 最新同日版本参与统计。
- 汇总按作品 ID 自动累加。
- 每日指标展示平台原值。
- 汇总比率由系统重新计算。
- 点击素材后再加载趋势和明细。
- 数据云端共享。
- 网站公开访问且任何人可管理。
- 采用简单的数据后台视觉风格。
